# Описание выполнения тестового задания

### Первое задание

   Оно выполенено на Python3, поэтому запускается так:
```python3 1.py```
   В скрипте используется генератор логина для подстановки в функцию.
   Он генерирует новый логин при каждом запуске.
   Результат выполнения программы выводится в консоль.
   
### Второе задание

   Перед запуском скрипта необходимо изменить права доступа при помощи
   команды `chmod +x 2.sh`
   Скрипт запускать там же, где запущен postgres, с помощью команды `./2.sh`
   
### Третье задание

   Если формат access.log нам известен, и мы знаем, что ip-адреса, 
   расположены, например в первом столбце, то можно воспользоваться 
   таким запросом

    `awk '{print $1}' access.log | sort | uniq -c | sort -r | head -n 10`

   Здесь:
   * awk - вызов утилиты awk
   * '{print $1}' - печать столбца с IP
   * access.log - обращение к файлу
   * sort - команда сортировки
   * uniq -c - команда для вывода количества повторений
   * sort -r - сортировка по убыванию
   * head -n 10 - вывод первых 10 строк

### Шестое задание

   Задача разбивается на следующие составляющие:
- необходимо считать строку, проверить введенные данные, распарсить ее на отдельные адреса и убедиться, что это адреса
- обратиться по каждому адресу и пропарсить его на наличие строки "Go". Посчитать количество совпадений, сохранить
- настроить параллельное выполнение функций

Если запуск горутины выглядит как go f(i), то настройка запуска нескольких горутин выглядит как:
```
    {
        for i := 0; i < 10; i++ {
        go f(i)
    }
```    
В нашем случае i должно быть ограничено количество поданных адресов, но не более 5 одновременно.


Импорт модулей происходит с помощью команды import, например
```  
import (
  "fmt"
  "net/http"
  "os"
  "strings"
)
```  
fmt - модуль форматированного ввода/вывода

strings - модуль для работы с UTF-8

net/http - модуль для работы с http клиентом/сервером и сетью

Для обеспечания работы разных гортин друг с другом используются каналы. 
Тип канала представлен ключевым словом chan, за которым следует тип, 
который будет передаваться по каналу. Оператор <- используется для 
отправки и получения сообщений по каналу. 

Структура даннов программы должна содержать:
 * функцию для парсинга url (с проверкой на ненулевую длину)
 * горутину для подсчета искомой строки
 * функцию, которая будет ограничивать одновременное количество горутин
